Viser: Imperative, Web-based 3D Visualization in Python

Brent Yi, Chung Min Kim, Justin Kerr, Gina Wu, Rebecca Feng, Anthony Zhang, Jonas Kulhanek, Hongsuk Choi, Yi Ma, Matthew Tancik, Angjoo Kanazawa
UC Berkeley · nerfstudio-project
3D Visualization Python Tooling Web-based Computer Vision Robotics Gaussian Splats
One-sentence problem: Existing 3D visualization tools for computer vision and robotics are either too rigid (desktop-only, hard to script), too heavy (full game engines), or too narrow (single primitive type) — how do we give researchers an easy, composable, SSH-accessible 3D viewer that works from any Python script or Jupyter notebook?
一句话问题：现有计算机视觉和机器人领域的 3D 可视化工具要么过于死板（仅限桌面、难以脚本化），要么过于庞大（完整游戏引擎），要么过于单一（仅支持单种图元）——如何为研究者提供一个简单、可组合、支持 SSH 访问的 3D 查看器，且能在任意 Python 脚本或 Jupyter Notebook 中直接使用？

Key Design Decisions & Contributions

  • Imperative Python API: State is set by direct assignment (server.scene.add_point_cloud(...)), not by declaring a scene graph. Fits naturally into existing research scripts — no boilerplate render loop required.
    命令式 Python API：通过直接赋值设置状态（server.scene.add_point_cloud(...)），而非声明场景图。自然融入已有研究脚本，无需样板渲染循环。
  • Web-based viewer architecture: The Python server streams scene updates over WebSockets; the client runs in any browser. Works over SSH tunnels out of the box — no GPU forwarding or X11 needed on remote machines.
    基于 Web 的查看器架构：Python 服务端通过 WebSocket 推送场景更新，客户端在任意浏览器中运行。开箱即用支持 SSH 隧道访问，远程服务器无需 GPU 转发或 X11。
  • Rich 3D + GUI primitive set: A comprehensive, composable library of 3D scene primitives and 2D GUI components that covers the vast majority of CV/robotics visualization needs without custom frontend code.
    丰富的 3D + GUI 图元集：覆盖绝大多数 CV/机器人可视化需求的全面可组合图元库，无需编写自定义前端代码。
  • Gaussian Splat support: First-class rendering of 3D Gaussian Splatting scenes directly in the browser — critical for the NeRF/3DGS research community (nerfstudio, gsplat).
    高斯泼溅支持：在浏览器中原生渲染 3D 高斯泼溅场景，对 NeRF/3DGS 研究社区（nerfstudio、gsplat）至关重要。
  • Multi-client & Jupyter integration: Multiple browser tabs can connect to the same server simultaneously. Full Jupyter notebook integration enables interactive visualization inline with analysis code.
    多客户端 & Jupyter 集成：多个浏览器标签页可同时连接同一服务端。完整 Jupyter Notebook 集成支持与分析代码内联的交互式可视化。

Architecture Overview

Python server (asyncio + WebSocket) ↔ Browser client (React + react-three-fiber / three.js + Mantine UI)

Server side (Python): An asyncio-based server manages scene state as a flat key-value store. Each add_*() call serializes a typed message (protobuf-style) and broadcasts it to all connected clients. Handles client-to-server events (clicks, GUI changes) via callbacks.

Client side (TypeScript): React SPA built with react-three-fiber for WebGL rendering, Mantine for UI components, and vanilla-extract for styling. Maintains a local scene graph synced from the server. Gaussian splat rendering uses a custom WebGL shader path.

Coordinate system: Uses right-handed, Y-up world coordinates with a consistent camera convention across all primitives, resolving a common pain point in CV tooling.

Transport: A binary WebSocket protocol with delta updates keeps bandwidth low even for large scenes. Reconnection and state replay are handled automatically.

Python 服务端（asyncio + WebSocket）↔ 浏览器客户端（React + react-three-fiber / three.js + Mantine UI）

服务端（Python）：基于 asyncio 的服务端将场景状态维护为平坦键值存储。每次 add_*() 调用将类型化消息序列化并广播给所有已连接客户端，通过回调处理客户端到服务端的事件（点击、GUI 变更）。

客户端（TypeScript）：使用 react-three-fiber 进行 WebGL 渲染、Mantine 构建 UI 组件、vanilla-extract 处理样式的 React 单页应用。维护与服务端同步的本地场景图，高斯泼溅渲染采用自定义 WebGL 着色器路径。

坐标系：采用右手系、Y 轴朝上的世界坐标，所有图元使用统一相机约定，解决了 CV 工具链中常见的坐标系混乱问题。

传输：基于增量更新的二进制 WebSocket 协议，即使场景较大也能保持低带宽占用。断线重连和状态重放均自动处理。

Primitive Library

3D Scene Primitives Point clouds · Meshes (file / programmatic) · Line segments & splines · Coordinate frames · Arrows · Gaussian splats · Batched mesh instances · Images as textures · Depth compositing · Lighting & shadows
2D GUI Primitives Buttons · Checkboxes · Sliders · Text inputs · Dropdowns · Folders / tabs · Markdown text · Modal dialogs · Notifications · Plotly / uPlot charts · Command palette · Transform gizmos · Click / hover callbacks
All primitives support hierarchical scene-graph parenting via wxyz / position arguments. GUI components support layout nesting (folders within folders) and theming.
所有图元均通过 wxyz / position 参数支持层级场景图父子关系。GUI 组件支持布局嵌套（文件夹套文件夹）和主题定制。

TL;DR

Viser is a Python 3D visualization library built around two design bets: an imperative API (just call add_*(), no scene-graph boilerplate) and a web-based viewer (runs in any browser, including over SSH). It ships a comprehensive set of 3D scene primitives (point clouds, meshes, Gaussian splats, coordinate frames…) plus rich 2D GUI widgets, making it the go-to tool for interactive visualization in NeRF/3DGS, SLAM, and egocentric CV research. 2.6k GitHub stars and actively maintained as of 2026.
