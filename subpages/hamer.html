HaMeR: Reconstructing Hands in 3D with Transformers

Georgios Pavlakos, Dandan Shan, Ilija Radosavovic, Angjoo Kanazawa, David Fouhey, Jitendra Malik
UC Berkeley · University of Michigan
Hand Regression MANO ViT In-the-Wild CVPR 2024
📄 arXiv ⭐ GitHub — geopavlakos/hamer
One-sentence problem: Can a fully transformer-based architecture, trained on a large and diverse dataset mix, significantly outperform CNN-based hand mesh recovery methods — especially on challenging in-the-wild occlusion scenarios?
一句话问题：完全基于 Transformer 的架构，在大规模多样化数据混合训练下，能否显著超越基于 CNN 的手部网格恢复方法——尤其是在具有挑战性的野外遮挡场景中？

Key Contributions

  • Fully transformer-based pipeline (ViT-H + transformer decoder): First hand mesh recovery system to replace CNN backbone with a large ViT-H, feeding patch tokens into a transformer decoder that regresses MANO parameters (θ ∈ ℝ⁴⁸, β ∈ ℝ¹⁰, translation t ∈ ℝ³).
    完全 Transformer 架构（ViT-H + Transformer 解码器）：首个将 CNN 主干替换为大型 ViT-H 的手部网格恢复系统，将图像块 token 送入 Transformer 解码器，回归 MANO 参数（θ ∈ ℝ⁴⁸，β ∈ ℝ¹⁰，位移 t ∈ ℝ³）。
  • Large-scale diverse training (2.7M samples from 10 datasets): Consolidates 7 datasets with 3D ground truth (FreiHAND, HO3D, MTC, RHD, InterHand2.6M, H2O3D, DexYCB) and 3 datasets with 2D keypoints (COCO WholeBody, Halpe, MPII NZSL), with adversarial loss to handle 2D-only supervision.
    大规模多样化训练（来自 10 个数据集的 270 万样本）：整合 7 个含 3D 真值的数据集和 3 个含 2D 关键点的数据集，通过对抗损失处理仅有 2D 监督的情况。
  • HInt — new in-the-wild evaluation benchmark: 40.4K hand annotations from Ego4D (23.2K), New Days/YouTube (12.0K), and EPIC-KITCHENS VISOR (5.3K), with first-ever occlusion labels per keypoint. 86.7% of hands in contact with objects or other hands.
    HInt——新的野外评估基准：来自 Ego4D（23.2K）、YouTube（12.0K）和 EPIC-KITCHENS VISOR（5.3K）的 40,400 个手部标注，是首个提供每关键点遮挡标签的数据集。86.7% 的手处于与物体或其他手的接触状态。
  • 2–3× improvement on in-the-wild benchmarks: On all three HInt splits, HaMeR surpasses prior SOTA (FrankMocap, HandOccNet) by 2–3× in PCK@0.05, with particularly large gains on occluded joints.
    野外基准 2–3 倍提升：在 HInt 三个子集上，HaMeR 的 PCK@0.05 比先前最优方法高 2–3 倍，在遮挡关节上提升尤为显著。

Method

RGB image (cropped hand) → ViT-H backbone (patch tokens) → Transformer decoder (1 output token, cross-attention) → MANO params (θ, β, t) → 778 vertices · 21 joints

Training losses (three objectives):

  • 3D supervision loss: ‖θ−θ*‖₂² + ‖β−β*‖₂² + ‖X−X*‖₁ — applied when 3D ground truth is available.
  • 2D reprojection loss: ‖x−x*‖₁ — for samples with only 2D keypoint annotations, projects predicted joints to image plane.
  • Adversarial loss: Σₖ(Dₖ(Θ)−1)² — discriminators on shape, pose, and per-joint angles enforce natural hand configurations under weak 2D supervision.

Training data composition: 2.7M total samples (4× more than FrankMocap). Only ~5% from in-the-wild sources; the HInt benchmark specifically tests generalization beyond lab data.

训练损失（三个目标）：

  • 3D 监督损失：有 3D 真值时优化 MANO 参数和关节位置的 L2/L1 损失。
  • 2D 重投影损失：仅有 2D 关键点标注时，将预测关节投影到图像平面计算 L1 损失。
  • 对抗损失：对形状、姿态和逐关节角度分别训练判别器，在 2D 弱监督下强制自然手部构型。

训练数据组成：共 270 万样本（FrankMocap 的 4 倍），其中仅约 5% 来自野外场景。HInt 基准专门测试对实验室数据之外的泛化能力。

Key finding: Data scaling and model capacity are both necessary — their combination yields multiplicative (not additive) gains. Neither alone closes the gap: large data with ResNet50 reaches 31.3%, large model with 25% data reaches 25.9%, but only the combination achieves 48.0%.
关键发现：数据规模扩展和模型容量都是必要的——两者结合带来乘法而非加法增益。仅扩展数据（ResNet50）达到 31.3%，仅增大模型（25% 数据）达到 25.9%，只有两者结合才能达到 48.0%。

Limitations

  • Single-frame processing: Each frame is processed independently — no temporal modeling; produces smooth video results empirically but not by design.
    单帧处理：每帧独立处理，无时序建模；实验上产生平滑视频结果，但并非设计所致。
  • No 3D in-the-wild evaluation: HInt only provides 2D labels; true 3D accuracy on unconstrained real-world data cannot be directly measured.
    无野外 3D 评估：HInt 仅提供 2D 标签，无法直接衡量真实场景下的 3D 精度。
  • Training data imbalance: ~95% controlled / ~5% in-the-wild — despite strong generalization, the model may still struggle with extreme domain shifts not covered by training data.
    训练数据失衡：约 95% 受控 / 5% 野外场景——尽管泛化能力强，模型仍可能在训练未覆盖的极端领域偏移下挣扎。
  • MANO model constraint: Output is constrained to MANO topology — cannot represent non-standard hand morphologies or interacting two-hand configurations directly.
    MANO 模型约束：输出受限于 MANO 拓扑，无法直接表示非标准手型或双手交互构型。

TL;DR

HaMeR is a ViT-H + transformer decoder pipeline for single-image 3D hand mesh recovery (MANO). Trained on 2.7M samples from 10 datasets with mixed 2D/3D supervision, it achieves SOTA on FreiHAND (PA-MPJPE 6.0mm) and HO3Dv2 (AUC 0.846), and delivers a 2–3× PCK improvement over prior methods on the new HInt in-the-wild benchmark (Ego4D + EPIC-KITCHENS + YouTube). The key insight is that data scale and model capacity both matter — combining ViT-H with full training data produces multiplicative gains, not additive ones.
