EMHI: A Multimodal Egocentric Human Motion Dataset with HMD and Body-Worn IMUs

One-sentence problem: Can combining head-mounted stereo vision with body-worn IMUs overcome the complementary failure modes of each modality — vision's self-occlusion and IMU's drift — to achieve accurate full-body SMPL pose estimation in egocentric VR/AR settings?
一句话问题：将头戴双目视觉与体穿式 IMU 结合，能否克服两种模态各自的缺陷——视觉的自遮挡和 IMU 的漂移——在自我中心 VR/AR 场景中实现精确的全身 SMPL 姿态估计？

Key Contributions

  • EMHI Dataset — first real-world synchronized HMD+IMU dataset: 885 sequences, 58 subjects (29M/29F), 39 common actions, ~28.5 hours, 3.07M synchronized data pairs at 30 FPS. Stereo egocentric images (640×480) from PICO4 headset + 5 body-worn IMUs (head, 2× hands, 2× legs). First real-world dataset to pair HMD stereo imagery with multi-body IMUs and SMPL-format ground truth.
    EMHI 数据集——首个真实同步 HMD+IMU 数据集：885 个序列，58 名受试者（29男/29女），39 种常见动作，约 28.5 小时，30 FPS 下 307 万同步数据对。PICO4 头显采集的双目自我中心图像（640×480）+ 5 个体穿式 IMU（头部、双手各一、双腿各一）。首个将 HMD 双目图像与多部位 IMU 及 SMPL 格式真值配对的真实世界数据集。
  • Rigorous GT annotation pipeline: Ground truth captured by 8-camera Kinect + Optitrack optical system (max sync deviation 16.5ms). Pipeline: HRNet 2D keypoints on multi-view RGB → triangulation → SMPL fitting with 5-term energy (rotation, joint, prior, smoothness, regularization constraints), with IMU rotations as additional constraints for occluded joints.
    严格的 GT 标注流程：真值由 8 路 Kinect + Optitrack 光学系统采集（最大同步偏差 16.5ms）。流程：HRNet 在多视角 RGB 上检测 2D 关键点 → 三角化 → 包含 5 项能量（旋转、关节、先验、平滑、正则约束）的 SMPL 拟合，并将 IMU 旋转作为遮挡关节的额外约束。
  • MEPoser baseline — multimodal fusion beats single-modal: Three-stage pipeline: multimodal fusion encoder (vision + IMU) → temporal LSTM encoder → SMPL regression head. MPJPE 3.7cm (Protocol 1), outperforming vision-only (+22% better) and IMU-only (+68% better). Validated against EgoPoseFormer, UnrealEgo, HMD-Poser.
    MEPoser 基线——多模态融合优于单模态：三阶段流水线：多模态融合编码器（视觉 + IMU）→ 时序 LSTM 编码器 → SMPL 回归头。MPJPE 3.7cm（协议 1），比纯视觉好 22%，比纯 IMU 好 68%。与 EgoPoseFormer、UnrealEgo、HMD-Poser 进行了对比验证。
  • Two evaluation protocols for cross-subject and cross-action generalization: Protocol 1 (unseen subjects, same actions) and Protocol 2 (unseen subjects + unseen actions) rigorously test generalization beyond training distribution.
    两种评估协议分别测试跨受试者和跨动作泛化：协议 1（未见受试者，相同动作）和协议 2（未见受试者 + 未见动作）严格测试超出训练分布的泛化能力。

EMHI Dataset Details

采集硬件

Egocentric capture: PICO4 VR headset with 2 downward-sloping stereo cameras (640×480, 30FPS) + 2 hand controllers + 2 leg trackers
5 IMUs: Head (6DoF), Left hand (6DoF), Right hand (6DoF), Left leg (3DoF), Right leg (3DoF)
GT capture: 8 Kinect cameras (RGB+depth) + Optitrack optical marker system
Sync precision: Max deviation ≤ 16.5ms, linear interpolation applied
自我中心采集：PICO4 VR 头显配 2 路向下倾斜双目相机（640×480，30FPS）+ 2 个手柄控制器 + 2 个腿部追踪器
5 个 IMU：头部（6DoF）、左手（6DoF）、右手（6DoF）、左腿（3DoF）、右腿（3DoF）
GT 采集：8 路 Kinect（RGB+深度）+ Optitrack 光学标记系统
同步精度：最大偏差 ≤ 16.5ms，应用线性插值

GT Annotation Pipeline
8-cam Kinect RGB → HRNet 2D keypoints (Body25) → Triangulation (smoothness + bone-length constraints) → 3D joints → SMPL fitting
E(θ,β) = λ_rot·E_rot + λ_joint·E_joint + λ_prior·E_prior + λ_smooth·E_smooth + λ_reg·E_reg
θ ∈ ℝ^75 (pose) · β ∈ ℝ^10 (shape) · IMU rotations constrain head & wrist angles → SMPL (θ, β) GT

Annotation quality validated via marker-based optical mocap: mean MPJPE 2.51cm across diverse actions (walking 2.61, hand-waving 2.72, boxing 2.73, taichi 2.98).

MEPoser Architecture

Stream A — Vision
Stereo images (640×480×1) × 2 → RegNetY-400MF backbone → {F_left, F_right} ∈ ℝ^(80×60×256) → Heatmap head (BCE loss, J=22 joints) + 3D MLP (local cam coords → global via headset 6DoF) → P̂_global^(J×3)
Stream B — IMU
5 IMU signals (head 6DoF + hands 6DoF×2 + legs 3DoF×2) → MLP encoder → IMU feature vector
Fusion
IMU feat + image feat + 3D joint feat → Concat + flatten → f^ti (multimodal fused feature per frame)
Temporal Encoder
{f^ti} → N=3 LSTM blocks (each: LSTM + FC-FFN + residual + LayerNorm) → temporally aggregated feature
SMPL Decoder
temporal feature → 2-layer MLP → θ ∈ ℝ^75 (pose) + β ∈ ℝ^10 (shape) → FK with headset 6DoF → 3D joint positions (world space)
Why vision + IMU is complementary: HMD stereo vision struggles with lower-body joints that are out of FOV (self-occluded by torso) — IMU leg trackers directly measure leg orientation. Conversely, IMU suffers from drift over time and cannot recover absolute 3D positions — visual features anchor the global pose. The fusion strategy (simple concat) lets the temporal LSTM learn which modality to trust per joint per frame.

Loss function: ℒ = λ_hp·ℒ_heatmap2D + λ_lj·ℒ_joints_local + λ_gj·ℒ_joints_global + λ_smpl·(ℒ_ori + ℒ_lrot + ℒ_grot + ℒ_joint) + λ_smooth·ℒ_smooth
为何视觉 + IMU 互补：HMD 双目视觉难以处理 FOV 之外的下肢关节（被躯干自遮挡）——IMU 腿部追踪器直接测量腿部朝向。反之，IMU 随时间累积漂移且无法恢复绝对 3D 位置——视觉特征为全局姿态提供锚点。融合策略（简单拼接）让时序 LSTM 学会对每帧每关节信任哪种模态。

损失函数：ℒ = λ_hp·ℒ_热图2D + λ_lj·ℒ_关节局部 + λ_gj·ℒ_关节全局 + λ_smpl·(ℒ_根朝向 + ℒ_局部旋转 + ℒ_全局旋转 + ℒ_关节) + λ_smooth·ℒ_平滑

Key finding — modality specialization: Vision is stronger on upper body (3.3 vs 5.0 for IMU) because the HMD cameras see arms and torso well. IMU is relatively weaker on lower body but provides direct rotation measurements that prevent the large errors seen in vision-only (6.3cm lower body). The fusion (3.7 / 2.7 / 5.1) improves all body parts, confirming complementarity rather than redundancy. Lower body error remains ~2× upper body even with fusion — head-mounted cameras fundamentally miss the legs.
关键发现——模态专业化：视觉在上半身更强（3.3 vs IMU 的 5.0），因为 HMD 相机能较好看到手臂和躯干。IMU 在下半身相对较弱，但提供直接的旋转测量，防止了纯视觉中出现的大误差（下半身 6.3cm）。融合（3.7 / 2.7 / 5.1）改善了所有身体部位，证实了互补性而非冗余性。即使融合后，下半身误差仍约为上半身的 2 倍——头戴相机从根本上无法拍到腿部。

TL;DR

EMHI (Pico, AAAI 2025) contributes two things: (1) the first real-world dataset synchronizing PICO4 HMD stereo video with 5 body-worn IMUs and SMPL GT (885 sequences, 58 subjects, 39 actions, 28.5 hrs, 3.07M pairs), and (2) MEPoser, a three-stage baseline (RegNetY fusion encoder → N=3 LSTM temporal encoder → 2-layer MLP SMPL decoder). Multimodal fusion (vision + IMU) achieves MPJPE 3.7cm on Protocol 1, outperforming vision-only (4.5cm, ↓18%) and IMU-only (6.2cm, ↓40%). Key limitation: lower-body error (~5.1cm) remains ~2× upper-body (~2.7cm) even with leg IMUs — headset cameras simply cannot see the legs. Output is full-body SMPL (θ ∈ ℝ^75, β ∈ ℝ^10), not SMPL-X.
