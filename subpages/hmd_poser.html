HMD-Poser: On-Device Real-time Human Motion Tracking from Scalable Sparse Observations

Peng Dai, Yang Zhang, Tao Liu, Zhen Fan, Tianyuan Du, Zhuo Su, Xiaozheng Zheng, Zeming Li
ByteDance Pico Inc.

📄 arXiv ⭐ GitHub 🌐 Project
One-sentence problem: How can we recover full-body SMPL motion from a VR headset in real-time on-device, using only head + hand tracking (and optionally 2–3 leg IMUs), with a single unified model that handles all sensor configurations?
一句话问题：如何仅使用头部 + 手部追踪（可选 2–3 个腿部 IMU），在 VR 头显上以单一统一模型实时恢复全身 SMPL 运动？

Key Contributions

  • Scalable sparse input: Single model handles 3 configurations via zero-padding — HMD-only (head + 2 hands), HMD+2IMUs (+ left/right leg), HMD+3IMUs (+ pelvis). No separate models needed.
    可扩展稀疏输入：单一模型通过零填充处理 3 种配置——仅 HMD（头部 + 双手）、HMD+2IMU（+ 左右腿）、HMD+3IMU（+ 骨盆）。无需独立模型。
  • Lightweight TSFL network: Temporal-Spatial Feature Learning = LSTM (per-component temporal modeling) + Transformer encoder (spatial correlations across M=8 components). 5× faster than pure Transformer baselines by reducing sequence dimension from M=40-45 to M=8.
    轻量 TSFL 网络：时序-空间特征学习 = LSTM（每组件时序建模）+ Transformer 编码器（M=8 组件间空间相关性）。将序列维度从 M=40-45 降至 M=8，比纯 Transformer 基线快 5 倍。
  • Online body shape estimation: Dedicated shape head regresses SMPL β online — eliminates "default body shape" assumption. Alone reduces MPJPE by 37.2% and H-PE by 61.2% compared to fixed-shape baselines.
    在线体型估计：专用形状头在线回归 SMPL β——消除"默认体型"假设。单独使 MPJPE 降低 37.2%，H-PE 降低 61.2%。
  • 90 FPS on PICO 4 HMD: Deployed commercially on standalone VR device; drives avatar in real-time for VR experiences. Includes new free-dancing evaluation dataset.
    PICO 4 头显上 90 FPS：在独立 VR 设备上商业部署；实时驱动 VR 体验中的虚拟形象。包含新的自由舞蹈评估数据集。

Architecture

Raw sensor data → Feature Embedding (135-dim input → higher-dim) → TSFL ×2 (LSTM temporal + Transformer spatial, M=8) → Dual MLP heads (pose θ + shape β) → SMPL FK → joint positions
Input configurations (zero-pad to 135-dim):输入配置（零填充至 135 维）：
HMD-only: head + 2 hands · 54-dim HMD+2IMUs: + left/right leg · 84-dim HMD+3IMUs: + pelvis · 99-dim
Input features per component: HMD/hands → position, linear velocity, 6D rotation, angular velocity; IMUs → 6D rotation, angular velocity, acceleration. Also adds relative hand-to-head representation (x^{l/h}_{h/t}, x^{r/h}_{h/t}) — critical for hand-joint accuracy (+30% H-PE improvement).

TSFL design insight: Pure Transformer on 40-45 frame sequences is expensive (O(M²d)). HMD-Poser uses LSTM to capture temporal dynamics per component first → hidden states become M=8 spatial tokens → Transformer encoder handles only spatial cross-component attention. Result: 205 FPS on GPU vs 2 FPS for AvatarJLM.
每组件输入特征：HMD/手部 → 位置、线速度、6D 旋转、角速度；IMU → 6D 旋转、角速度、加速度。还增加手部相对头部的表示——对手关节精度至关重要（+30% H-PE 提升）。

TSFL 设计洞察：对 40-45 帧序列的纯 Transformer 计算昂贵（O(M²d)）。HMD-Poser 先用 LSTM 捕获每组件时序动态 → 隐状态成为 M=8 个空间 token → Transformer 编码器仅处理空间跨组件注意力。结果：GPU 上 205 FPS vs AvatarJLM 的 2 FPS。

Limitations

  • Sim-to-real gap: Trained on synthetic AMASS data; real PICO 4 sensor noise degrades MPJPE from 4.75cm (synthetic input) to 6.55cm (real sensors) — a 38% accuracy drop.
    仿真到真实的差距：在合成 AMASS 数据上训练；真实 PICO 4 传感器噪声使 MPJPE 从 4.75cm（合成输入）降至 6.55cm（真实传感器）——精度下降 38%。
  • IMU lower-body ambiguity: Struggles with slowly lifted legs (similar IMU readings regardless of which foot) — inherent limitation of sparse sensors.
    IMU 下肢歧义：难以区分缓慢抬腿动作（两脚的 IMU 读数相似）——稀疏传感器的固有限制。
  • No video input: Purely sensor-driven — no RGB information means no scene context, hand-object interaction, or visual grounding. Complementary to video-based methods like EgoAllo.
    无视频输入：纯传感器驱动——无 RGB 信息意味着没有场景上下文、手-物体交互或视觉锚定。与 EgoAllo 等基于视频的方法互补。

TL;DR

HMD-Poser (ByteDance Pico, CVPR 2024) recovers full-body SMPL motion from sparse VR sensors (HMD + optional leg IMUs) in real-time on a standalone VR headset (90 FPS on PICO 4). Key innovations: (1) scalable zero-padding for unified multi-config handling; (2) LSTM+Transformer TSFL — 5× faster than pure Transformer at same accuracy; (3) online body shape head — the biggest gain, ↓37% MPJPE. On AMASS HMD-only: MPJPE 3.19cm vs 5.03cm (AvatarJLM). Sensor-driven, no RGB — complementary to egocentric video methods like EgoAllo.
