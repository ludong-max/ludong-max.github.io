EgoPoseFormer v2: Accurate Egocentric Human Motion Estimation for AR/VR

Zhenyu Li, Sai Kumar Dwivedi, Filip Maric, Carlos Chacón, Nadine Bertsch, Filippo Arcadu, Tomas Hodan, Michael Ramamonjisoa, Peter Wonka, Amy Zhao, Robin Kips, Cem Keskin, Anastasia Tkach, Chenhongyi Yang

Keypoint XYZ / Body Model Momentum Body Model Egocentric Transformer Semi-Supervised Real-time AR/VR CVPR 2026 Meta
📄 arXiv
One-sentence problem: How can we build a fully end-to-end, temporally-consistent egocentric body pose estimator for AR/VR that supports both 3D keypoint (XYZ) and parametric body model (Momentum, joint rotations + FK) outputs, and also scales to millions of unlabeled frames via semi-supervised auto-labeling — all under a constant 0.8ms GPU latency budget?
一句话问题：如何构建一个完全端到端、时序一致的自我中心体态估计器，同时支持 3D 关键点（XYZ）和参数化体态模型（Momentum，关节旋转 + FK）两种输出，并通过半监督自动标注扩展到数百万帧无标注数据——且 GPU 延迟保持恒定 0.8ms？

Two Key Contributions

① End-to-End Transformer Architecture

Single holistic pose query + two stacked transformer decoders with conditioned multi-view cross-attention (projection-based, hardware-friendly) + causal temporal attention (RoPE + KV-Cache). Fully differentiable, constant compute regardless of body representation type.
单一整体姿态查询 + 两个堆叠 Transformer 解码器，含条件化多视角交叉注意力（基于投影，硬件友好）+ 因果时序注意力（RoPE + KV-Cache）。完全可微，计算量不随体态表示类型变化。

② Auto-Labeling System (ALS)

Teacher (larger ViT-B/DINOv3) → pseudo-labels for 70M unlabeled frames. Student trained with labeled + pseudo-labeled + uncertainty distillation loss. Improves wrist MPJPE by 13.1%; scales to lightweight models (MobileNetV4-S benefits more than ResNet-18).
教师模型（ViT-B/DINOv3）为 7000 万帧无标注数据生成伪标签。学生模型在有标注 + 伪标注 + 不确定性蒸馏损失上训练。腕部 MPJPE 提升 13.1%；可扩展到轻量模型（MobileNetV4-S 受益更多）。

Architecture

Input → Pose Query Initialization Multi-view images {I^v} + headset H_t → Per-view encoder (ResNet-18 / ResNet-50 / ViT-B) → Single holistic query q_t = MLP(H_t) — identity + headset rotation conditioned
Decoder Block 1 — Coarse Pose Proposal Conditioned MV Cross-Attention (view identity σᵛ) + Causal Temporal Attention (RoPE, window w, KV-Cache) → Coarse 3D keypoints P̄
Decoder Block 2 — Pose Refinement Conditioned MV Cross-Attention (σᵛ + projected 2D keypoints o^v) + Causal Temporal Attention → Refined pose P_r + uncertainty Σ (per-keypoint 6D covariance)

Key innovation vs EPFv1: EPFv1 used one separate Joint Query Token (JQT) per keypoint — computation scaled with J joints, no end-to-end gradient flow to the proposal stage, and deformable attention made edge deployment hard. EPFv2 uses a single holistic query conditioned on headset pose/identity, making computation constant, fully E2E trainable, and hardware-friendly (standard MHA instead of deformable attention).

Projection-conditioned cross-attention: Coarse 3D keypoints are projected onto each camera plane → 2D coordinates → Condition Positional Encoding (CPE) appended to query. This gives geometric spatial anchoring without expensive deformable modules.

vs EPFv1 关键改进：EPFv1 每个关键点用一个独立的 Joint Query Token，计算随关节数线性增长，Proposal 阶段无端到端梯度流，可形变注意力难以边缘部署。EPFv2 使用单一整体查询（由头显姿态/身份条件化），计算恒定，完全端到端，且用标准 MHA 替代可形变注意力，硬件友好。

投影条件化交叉注意力：粗略 3D 关键点投影到各相机平面 → 2D 坐标 → 作为条件位置编码（CPE）附加到查询。无需昂贵的可形变模块即可实现几何空间锚点。

EgoBody3M 标注细节

Raw GT = 3D keypoints only (from outside-in MoCap). Body model parameters (joint rotations + body scales) are not directly captured — they are computed post-hoc by running Inverse Kinematics (IK) on the GT keypoints using Meta's Momentum body tool (facebookresearch.github.io/momentum). This means the parametric supervision signal is derived, not directly observed.
原始 GT = 仅有 3D 关键点（来自外部 MoCap）。Body model 参数（关节旋转 + body scales）并非直接采集——而是通过对 GT 关键点跑 IK（逆运动学） 经由 Meta 的 Momentum 工具（facebookresearch.github.io/momentum）事后计算得到。也就是说参数化监督信号是推导出的，而非直接观测的。

EGO-ITW-70M 与 ALS

Meta-private, no public release. Collected with the same 4-camera VR headset hardware as EgoBody3M. No ground-truth labels at all. Used exclusively as unlabeled corpus for the Auto-Labeling System (ALS): teacher (ViT-B / DINOv3) → pseudo-labels → uncertainty distillation → student. Fig. 6 shows ALS benefit saturates around 50–100% of 70M frames; MobileNetV4-S benefits proportionally more than ResNet-18.
Meta 私有，不公开。与 EgoBody3M 使用相同的 4 相机 VR 头显硬件采集。完全无 GT 标注。仅用于 ALS 的无标注语料：教师模型（ViT-B / DINOv3）→ 伪标签 → 不确定性蒸馏 → 学生模型。Fig. 6 显示 ALS 收益在 50–100% 的 70M 帧附近趋于饱和；MobileNetV4-S 受益比例更大。
⚠️ Reproducibility caveat: EGO-ITW-70M is a Meta-internal dataset — external researchers cannot replicate the full ALS pipeline. Only the EgoBody3M-based results (w/o ALS, MPJPE 4.17cm) are fully reproducible from public data.
⚠️ 可复现性说明：EGO-ITW-70M 是 Meta 内部数据集——外部研究者无法复现完整的 ALS 流程。仅基于 EgoBody3M 的结果（w/o ALS，MPJPE 4.17cm）可用公开数据完全复现。

Limitations

  • Multi-camera dependency: Requires 4 synchronized cameras — single-camera setup not evaluated. Limits deployment to purpose-built VR headsets like Meta Quest.
    多相机依赖：需要 4 个同步相机——未评估单相机配置。限制了部署范围，仅适用于 Meta Quest 等专用 VR 头显。
  • EgoBody3M-centric evaluation: All benchmarks on EgoBody3M (Meta's proprietary dataset). Generalization to other devices (Aria, HoloLens) shown only qualitatively on XR-MBT.
    以 EgoBody3M 为中心的评估：所有基准测试均在 EgoBody3M（Meta 专有数据集）上进行。跨设备泛化（Aria、HoloLens）仅在 XR-MBT 上进行定性展示。
  • No world-space output: Outputs in headset/camera space — not allocentric world coordinates (unlike EgoAllo). Feet/legs still have highest error (~8.7cm).
    无世界坐标系输出：输出在头显/相机坐标系下——非异中心世界坐标（与 EgoAllo 不同）。脚部/腿部误差仍最高（~8.7cm）。
  • ALS requires large unlabeled corpus: 70M frames needed for meaningful improvement; collecting hardware-matched unlabeled data at scale may not be feasible for all research groups.
    ALS 需要大量无标注数据：需要 7000 万帧才能获得显著提升；并非所有研究团队都能大规模采集硬件匹配的无标注数据。

TL;DR

EgoPoseFormer v2 (Meta + KAUST, CVPR 2026) advances egocentric body pose estimation for AR/VR with two innovations: (1) a single holistic query transformer with projection-conditioned multi-view cross-attention + causal temporal attention (RoPE + KV-Cache), fully E2E at constant 0.8ms; (2) an uncertainty-aware auto-labeling system scaling to 70M unlabeled frames via teacher-student distillation. Output representation: supports both direct 3D keypoint XYZ regression AND parametric pose (joint rotations + body scales via Meta's Momentum body model → FK) — NOT SMPL-X. On EgoBody3M: MPJPE 4.02cm (↓22.4% vs EgoBody3M, ↓15.4% vs EPFv1) and temporal jitter ↓51.7%. ALS alone improves wrist MPJPE by 13.1%.
