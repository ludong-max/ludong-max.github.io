EgoPoseFormer: A Simple Baseline for Stereo Egocentric 3D Human Pose Estimation

Chenhongyi Yang1,2*, Anastasia Tkach2, Shreyas Hampali2, Linguang Zhang2, Elliot J. Crowley1, Cem Keskin2
1 University of Edinburgh  ·  2 Meta Reality Labs
Egocentric Pose Stereo Vision DETR-style Two-Stage VR/AR ECCV 2024
📄 ECCV PDF ⭐ GitHub — ChenhongyiYang/egoposeformer
One-sentence problem: How can stereo head-mounted cameras estimate the full 3D body pose when most joints are invisible — either out of the limited FOV or self-occluded by the torso?
一句话问题：当大多数关节因头戴相机视野受限或被躯干自遮挡而不可见时，如何从双目头戴相机估计完整的 3D 人体姿态？

Key Contributions

  • EgoPoseFormer — two-stage coarse-to-fine pipeline: A Pose Proposal Network (PPN) first computes a coarse body pose from globally-pooled stereo features via a 2-layer MLP; a DETR-style Pose Refinement Transformer (PRFormer) then refines the proposal using fine-grained multi-view stereo features.
    EgoPoseFormer——两阶段粗到精流水线：姿态提议网络（PPN）通过 2 层 MLP 从全局池化的双目特征中计算粗略体姿；DETR 风格的姿态精化 Transformer（PRFormer）再利用细粒度多视角立体特征对提议进行精化。
  • Deformable Stereo Attention: Replaces standard cross-attention in each PRFormer layer — projects each 3D joint proposal onto each view plane using camera parameters, applies deformable attention per view, then fuses all views via concat + linear projection. Enables precise multi-view stereo reasoning in a plug-and-play manner.
    可变形立体注意力：替换每个 PRFormer 层中的标准交叉注意力——利用相机参数将每个 3D 关节提议投影到各视平面，对每个视图应用可变形注意力，再通过拼接+线性投影融合所有视图。以即插即用方式实现精确的多视角立体推理。
  • Surprisingly strong PPN baseline: The simple MLP-based first stage alone already outperforms previous state-of-the-art methods on UnrealEgo — because global features capture invisible joint locations by reasoning jointly over visible joints and background context.
    出人意料的强 PPN 基线：仅凭简单的基于 MLP 的第一阶段就已在 UnrealEgo 上超越之前的最优方法——因为全局特征通过联合推理可见关节和背景上下文来捕获不可见关节的位置。
  • SOTA on UnrealEgo with extreme efficiency: MPJPE 33.4mm (45% improvement over Ego3DPose at 60.8mm) using only 7.9% of Ego3DPose's parameters (14.1M vs 178.4M) and 13.1% of its FLOPs (7.3G vs 55.6G).
    UnrealEgo 最优且极高效：MPJPE 33.4mm（比 Ego3DPose 的 60.8mm 提升 45%），仅使用 Ego3DPose 7.9% 的参数量（14.1M vs 178.4M）和 13.1% 的浮点计算量（7.3G vs 55.6G）。
  • Seamless monocular extension — SOTA on SceneEgo: MPJPE 93.0mm, improving 25.5mm (21%) over SceneEgo's own method (118.5mm) without requiring ground-truth depth or semantic masks, using only 60.7% of params and 32% of FLOPs.
    无缝扩展到单目——SceneEgo 最优：MPJPE 93.0mm，比 SceneEgo 方法（118.5mm）提升 25.5mm（21%），无需真实深度或语义掩码，仅需 60.7% 的参数量和 32% 的浮点计算量。

Method Architecture

Feature Extractor (shared)
Multi-view fisheye images → UNet (ResNet-18/50) pretrained on 2D heatmaps · per-view, no cross-view concat in decoder → F ∈ ℝ^(V×H×W×C)
Stage 1 — Pose Proposal Network (PPN)
F → GlobalAvgPool per view → Concat → 2-layer MLP (GELU) → coarse pose P⁰ ∈ ℝ^(Nⱼ×3)
Stage 2 — Pose Refinement Transformer (PRFormer, S=3 layers)
P⁰ + joint index σⱼ → JQT Generation MLP → Joint Query Tokens Q ∈ ℝ^(Nⱼ×C) → PRFormer Layers [Deformable Stereo Attn → Self-Attn → FFN → ΔP^s] → final pose P^S = P⁰ + ΔP^S
Deformable Stereo Attention key insight: For each JQT Qⱼ with 3D proposal P⁰ⱼ, the 3D point is projected to 2D reference coordinates on each view plane using known camera parameters. Deformable attention then samples local features around these reference points in each view independently. The per-view results Z_j^k are concatenated and linearly projected back to the original dimension — making the operation an atomic drop-in replacement for standard cross-attention. When a joint is outside the FOV of a view, the result for that view is zeroed out, giving the model an explicit signal of invisibility.
可变形立体注意力关键思路：对于带有 3D 提议 P⁰ⱼ 的每个 JQT Qⱼ，利用已知相机参数将 3D 点投影到各视平面的 2D 参考坐标。可变形注意力随后在每个视图中独立地在参考点周围采样局部特征。各视图的输出 Z_j^k 拼接后线性投影回原始维度——该操作是标准交叉注意力的原子级替换。当某关节超出某视图的视野时，该视图的结果置零，为模型提供明确的不可见信号。

Implementation Details — 核心实现细节

① Feature Extractor
Backbone: UNet with ResNet-18 (UnrealEgo) or ResNet-50 (SceneEgo). Feature downsampling stride = 4.
Key difference vs UnrealEgo baseline: Cross-view feature concatenation is removed from the UNet decoder — each view is encoded independently. This alone significantly reduces parameters and FLOPs because the PRFormer handles multi-view fusion downstream.
Pre-training: The feature extractor is first pre-trained on 2D joint heatmap prediction using a lightweight fully-convolutional head (monocular images → 2D heatmaps). The heatmap head is removed after pre-training. This gives the features spatially-aware joint priors before the 3D pipeline begins.
Output: F ∈ ℝ^(V × H/4 × W/4 × C) — per-view feature maps at 1/4 resolution.
主干：UNet + ResNet-18（UnrealEgo）或 ResNet-50（SceneEgo），特征下采样步长 = 4。
与 UnrealEgo 基线的关键区别：从 UNet 解码器中去除了跨视图特征拼接——每个视图独立编码。这一改动本身就显著减少了参数量和浮点计算量，因为多视图融合交由下游的 PRFormer 处理。
预训练：特征提取器首先在 2D 关节热图预测 任务上预训练，使用轻量全卷积头（单目图像 → 2D 热图），预训练后热图头被移除。这赋予特征在 3D 流水线开始前的空间感知关节先验。
输出：F ∈ ℝ^(V × H/4 × W/4 × C)——1/4 分辨率的逐视图特征图。
② Pose Proposal Network (PPN) — Stage 1
Input: Multi-view feature maps F.
Step 1 — Global pooling: Apply GlobalAvgPool to each view's feature map independently → per-view global descriptor.
Step 2 — Concat: Concatenate all V per-view descriptors into one unified feature vector.
Step 3 — MLP: 2-layer MLP with GELU activation → outputs coarse 3D locations for all Nⱼ joints simultaneously.
Formula: P⁰ = MLP_ppn( Concat_k( AvgPool(F_k) ) )
Why it works for invisible joints: Global pooling discards spatial locality — the whole scene context (visible joints, background, silhouettes) is encoded in a single vector. The MLP learns to infer invisible joint positions from this global context and human kinematic priors, similar to heatmap-based auto-encoder methods but simpler and more direct.
Surprising finding: This PPN alone (without PRFormer) already outperforms all previous SOTA on UnrealEgo.
输入：多视图特征图 F。
第一步——全局池化：对每个视图的特征图独立做 GlobalAvgPool → 每视图全局描述符。
第二步——拼接：将所有 V 个视图的全局描述符拼接为一个统一特征向量。
第三步——MLP：2 层 MLP + GELU 激活 → 同时输出所有 Nⱼ 个关节的粗略 3D 位置。
公式：P⁰ = MLP_ppn( Concat_k( AvgPool(F_k) ) )
为何对不可见关节有效：全局池化丢弃了空间局部性——整个场景上下文（可见关节、背景、轮廓）被编码进单一向量。MLP 从这个全局上下文和人体运动学先验中推断不可见关节的位置，类似热图自编码器方法，但更简洁直接。
惊人发现：仅此 PPN（无 PRFormer）已在 UnrealEgo 上超越所有之前的最优方法。
③ Joint Query Tokens (JQT) + PRFormer — Stage 2
JQT generation: Each joint j gets a unique query token by embedding its index σⱼ (used as a scalar joint identifier, simply set to joint index in practice) and its initial 3D location P⁰ⱼ = (x⁰,y⁰,z⁰) into a shared MLP:
Qⱼ = MLP_JQT( σⱼ, x⁰ⱼ, y⁰ⱼ, z⁰ⱼ ) This encodes both which joint (identity / kinematic role) and where it is (coarse location) into the query — inspired by DETR object queries.

PRFormer structure: S = 3 identical transformer decoder layers. Each layer s:
  1. Deformable Stereo Attention (cross-attention with multi-view stereo features)
  2. Self-Attention across all Nⱼ JQTs — encodes human kinematic dependencies (e.g., if elbow is visible, shoulder can be inferred)
  3. FFN — non-linear transform of each JQT
  4. Offset head: ΔPⱼˢ = MLP_offset( Qⱼˢ )

Final pose: Pˢ = ΔPˢ + P⁰ — always an offset relative to the PPN proposal, never absolute regression. This prevents drift and keeps gradients well-conditioned.
JQT 生成：每个关节 j 通过将其索引 σⱼ（实践中直接用关节索引作为标量标识符）和初始 3D 位置 P⁰ⱼ = (x⁰,y⁰,z⁰) 输入共享 MLP 生成唯一查询 token：
Qⱼ = MLP_JQT( σⱼ, x⁰ⱼ, y⁰ⱼ, z⁰ⱼ ) 这同时编码了是哪个关节（身份/运动学角色）和在哪里（粗略位置）——受 DETR 目标查询启发。

PRFormer 结构：S = 3 个相同的 Transformer 解码器层，每层 s：
  1. 可变形立体注意力（与多视图立体特征的交叉注意力）
  2. 所有 Nⱼ 个 JQT 之间的自注意力——编码人体运动学依赖（如肘部可见则可推断肩部）
  3. FFN——每个 JQT 的非线性变换
  4. 偏移头：ΔPⱼˢ = MLP_offset( Qⱼˢ )

最终姿态：Pˢ = ΔPˢ + P⁰——始终是相对于 PPN 提议的偏移，而非绝对回归。这防止了漂移并保持梯度的良好条件。
④ Deformable Stereo Attention — 逐步拆解
For joint j with JQT Qⱼ and proposal P⁰ⱼ, across V views:

Step 1 — 3D→2D projection (geometry-aware reference):
  Project P⁰ⱼ onto view k using known camera intrinsics/extrinsics → {P̃ⱼᵏ} = {(x̃ⱼᵏ, ỹⱼᵏ)} (2D pixel coordinates as reference point)

Step 2 — Per-view deformable attention:
  Zⱼᵏ = DeformAttn( Qⱼ, P̃ⱼᵏ, Fₖ )
  Deformable attention learns offsets from the reference point P̃ⱼᵏ to sample nearby feature locations — more expressive than fixed-grid sampling, handles appearance variation around the expected joint location.
  If joint j is outside the FOV of view k → Zⱼᵏ = 0 (explicit out-of-FOV signal).

Step 3 — Multi-view fusion:
  Zⱼ = Linear( Concat_k(Zⱼᵏ) )
  Concatenate all V view results → linear projection back to C dimensions. The linear layer learns to weight views by reliability (e.g., suppress out-of-FOV zeros, amplify informative views).

Why this replaces standard cross-attention: Standard cross-attention attends over the entire feature map globally and has no geometric grounding. Deformable Stereo Attention uses the coarse 3D proposal to focus on geometrically-consistent image regions across views — crucial for accurate 3D localization from multi-view stereo.
对于关节 j（JQT 为 Qⱼ，提议为 P⁰ⱼ），跨 V 个视图：

第一步——3D→2D 投影（几何感知参考点）：
  用已知相机内参/外参将 P⁰ⱼ 投影到视图 k → {P̃ⱼᵏ} = {(x̃ⱼᵏ, ỹⱼᵏ)}（2D 像素坐标作为参考点）

第二步——逐视图可变形注意力：
  Zⱼᵏ = DeformAttn( Qⱼ, P̃ⱼᵏ, Fₖ )
  可变形注意力从参考点 P̃ⱼᵏ 出发学习偏移量来采样附近特征位置——比固定网格采样更具表达力，能处理关节预期位置附近的外观变化。
  若关节 j 超出视图 k 的视野 → Zⱼᵏ = 0（显式的 FOV 外信号）。

第三步——多视图融合：
  Zⱼ = Linear( Concat_k(Zⱼᵏ) )
  拼接所有 V 个视图结果 → 线性投影回 C 维。线性层学会按可靠性对视图加权（如抑制 FOV 外的零值，放大信息量大的视图）。

为何替换标准交叉注意力：标准交叉注意力全局关注整个特征图，无几何锚点。可变形立体注意力利用粗略 3D 提议聚焦于跨视图几何一致的图像区域——对于多视角立体精确 3D 定位至关重要。
⑤ Loss Function & Training Recipe
Loss — per-stage per-joint L2 (auxiliary supervision on all S+1 stages):
ℒ = Σ_{s=0}^{S} Σ_{j=1}^{Nⱼ} ‖Pⱼˢ − P̂ⱼ‖₂ s=0 supervises the PPN proposal directly; s=1,2,3 supervise each PRFormer layer output. This deep supervision forces every intermediate prediction to be meaningful, not just the final output — key for training stability and the strong PPN baseline.

Ablation Study

Ablations on UnrealEgo study the two core components of PRFormer:
在 UnrealEgo 上的消融实验研究了 PRFormer 的两个核心组件：
  • PPN alone beats all prior SOTA: The simple 2-layer MLP pose proposal already surpasses all previous methods on UnrealEgo, demonstrating that global feature reasoning is the key missing ingredient in prior heatmap-based approaches.
    PPN 单独超越所有先前最优：简单的 2 层 MLP 姿态提议已在 UnrealEgo 上超越所有先前方法，表明全局特征推理是先前基于热图方法中缺少的关键要素。
  • Deformable Stereo Attention vs standard cross-attention: Replacing the proposed Deformable Stereo Attention with standard cross-attention significantly degrades performance — standard cross-attention cannot efficiently process fine-grained multi-view stereo features for accurate 3D joint localization.
    可变形立体注意力 vs 标准交叉注意力：将所提出的可变形立体注意力替换为标准交叉注意力会显著降低性能——标准交叉注意力无法高效处理细粒度多视角立体特征以实现精确的 3D 关节定位。
  • Self-attention in PRFormer: Removing the self-attention block (which encodes human kinematic information and joint-to-joint dependencies) also hurts performance, confirming that body structure context is complementary to stereo visual features.
    PRFormer 中的自注意力：移除自注意力块（编码人体运动学信息和关节间依赖关系）同样会损害性能，确认身体结构上下文与立体视觉特征是互补的。
  • Feature extractor design: Removing cross-view concatenation from the UNet decoder (compared to UnrealEgo's original design) significantly reduces parameters and FLOPs without hurting accuracy — the PRFormer handles multi-view fusion instead.
    特征提取器设计：从 UNet 解码器中去除跨视图拼接（与 UnrealEgo 原始设计相比）在不影响精度的情况下显著减少参数量和浮点计算量——多视图融合改由 PRFormer 处理。

TL;DR

EgoPoseFormer (ECCV 2024, Edinburgh + Meta) proposes a simple two-stage transformer pipeline for stereo egocentric 3D body pose estimation. Stage 1: a 2-layer MLP (PPN) uses globally-pooled stereo features to propose coarse joint locations — already beating all prior SOTA on its own. Stage 2: a DETR-style PRFormer with novel Deformable Stereo Attention refines proposals using fine-grained per-view image features referenced by projected 3D joint locations. On UnrealEgo: MPJPE 33.4mm (↓45%) with only 14.1M params / 7.3G FLOPs. Extends to monocular SceneEgo: MPJPE 93.0mm (↓21%) without GT depth or masks. Superseded by EgoPoseFormer v2 (CVPR 2026) which adds temporal modeling, single holistic query, and support for parametric body model output (Meta Momentum model, joint rotations + FK) — not SMPL-X.
